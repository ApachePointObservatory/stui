STUI:Version History

Compatibility

  • Mac: works with MacOS X 10.6, 10.5 and 10.4 and is a universal binary.
  • Unix: runs from source, so is compatible with all flavors of unix. See the installation instructions for the required version of Python and other libraries.
  • No Windows releases are planned.

Version 1.0b1-post 2010-07-21

Visible changes:

  • If you lose your connection or put your computer to sleep for long enough, TUI will test the connection and, if good, will attempt to refresh the data.

Bug fixes:

  • Log filtering based on command number would not show the outgoing command.
  • Would fail to start on unix due to an error in determining the name of the status window.

Version 0.5.3 2010-05-06

Visible changes:

  • Ticket 794 Add a 10 um selection to the stui Secondary Focus gui.

Bug fixes:

  • Ticket 688 "None" missing from filter options in log window.
  • Ticket 800 Windows are now displayed within the visible screen area, even if the saved positions is not (as can easily happen if you sometimes connect a laptop to an external monitor). The code assumes your visible screen is a rectangle, so it can be fooled by strange screen arrangements. (This fix is actually in RO 2.3.5, not in STUI).
  • Corrected the display of some MCP axis controller status bits:
    • Corrected "Pos. error too large to correct" from bit 17 to 24 and also changed it from a warning to an error so that it will display in red instead of blue.
    • Added error "Clock not set" bit 18 (formerly this bit was mislabeled as a windscreen touch).
    • Corrected warning "Windscreen touch" bits from 18 & 19 to 29 & 30.
    • Removed error bit 10, which not used but was listed as "A/D converter problem".
  • Traceback when design hour angle known but hour angle unknown.

Version 0.5.2 2010-03-19

Visible changes:

  • Overhauled the status window's display of instrument and cartridge information.
  • The About TUI window now shows the paths to special files such as preferences and TUIAdditions directories.
  • On unix, the shared TUIAdditions directory is now up one level (to the parent of the parent of the directory named TUI), so that it is independent of the TUI source tree and is shared by all versions of TUI.

Internal changes:

  • Updated the actor dictionaries
  • TUI.TUIPaths: added functions getGeomFile and getPrefsFile and added ifExists argument to getAddPaths.

Version 0.5.1 2010-03-12

Visible changes:

  • The log window is now visible by default.
  • The alerts window always shows disable alert rules.
  • Scripting tutorial and associated tutorial scripts rewritten for STUI.

Bug fixes:

  • Ticket 631 Paused timer has wrong "sign".

Internal changes:

  • Added TUI.Models.getModel(actor). This makes all existing models available and is this is the recommended way to get a model. The old technique will result in a deprecation warning for now, and will eventually stop working altogether.

Version 0.5.0 2010-03-11

Warning for unix users:

  • Requires RO Package version 2.3.4 and opscore trunk rev 107768 (the latter fixes some bugs in the script system).

Visible changes:

  • The Status window now includes information about the currently mounted plug-plate cartridge.
  • The Users window now shows each client, version and system info.

Bug fixes:

  • Ticket 613 STUI guider plate view background levels are wrong.
  • Save As is broken in The Run_Commands script.
  • Command replies could be shown in the wrong color in status bars (fix adapted from TUI).

Version 0.4 2010-02-19

Visible changes:

  • Added a Guider window.
  • Added a guider Focus Plot window.
  • Ticket 392 Changed log timestamps and time in the Status window from UTC to TAI.
  • Added a "Down Instrument" button to the Alerts window.
  • Improved the reliability of sound cues. Warning for unix users: this was done by switching from the tkSnack to pygame sound library. This will require you to install pygame (and its dependencies) and a new version of RO. See the unix installation instructions for more information.
  • Renamed STUI's Special Files as follows. This allow one to run TUI and STUI on the same computer without collisions. You must rename your files to restore your saved settings and additions.
    • TUIPrefs to STUIPrefs
    • TUIGeom to STUIGeom
    • TUIAdditions to STUIAdditions.

Bug fixes:

  • Ticket 584 STUI Run_Commands script does not work. In fact the entire scripting system was broken due to relying on the dispatcher, keyword variables and command variables from RO instead of opscore. Fixed by adding ScriptRunner to opscore.actor and ScriptWdg to TUI.Base.Wdg.
  • Much sample code did not work because it was not compatible with Twisted.

Version 0.3.1 2009-10-09

Visible changes:

  • Added a rudimentary BOSS status display.
  • New icon from a photo of the 2.5m telescope by Dan Long.

Bug fixes:

  • Stopped playing the Critical Alert sound cue for warning alerts.

Version 0.3 2009-09-16

Visible changes:

  • Added preliminary guider controls (Inst>Guide). Limitations include:
    • No way to enable or disable specific guide probes.
    • Image download and display is untested (because the guider actor is not yet outputting the files keyword).
    • Image plate view is disabled because the the necessary metadata has not yet specified or supplied.
  • Switched to new sound cues by Elena and Dmitry for critical and serious alerts. Note that warning alerts are still silent by default (by design).
  • Improved the html help documentation, including removing outdated references to the 3.5m.
  • Changed name from TUISDSS to STUI.

Bug fixes:

  • Restored the missing Axis Halt sound preference.

Version 0.2.1 2009-09-02

Visible changes:

  • Added the potential for separate sound cues for critical, serious and warning alerts. However, at present the defaults are:
    • No sound for warning alerts. Observers should comment on whether this is a good choice.
    • SeriousAlert.wav is used for both serious and critical sound cues. This will be changed when I have time to work on sound cues or am given a good pair of cues for serious and critical alerts.
  • Added a new color for critical alerts (defaults to orange).
  • Added help pages for the Alerts, Interlocks and MCP windows.

Warning for those running from source (rather than the Mac binary):

  • This version of STUI requires RO 2.2.19.
  • Older versions of STUI will not work with RO 2.2.19. (This is due to a glitch in the log window that I have fixed in STUI 0.2.1.)

Version 0.2 2009-08-25

Visible changes:

  • Added a preference to disable sound cues.

Warning for those running from source (rather than the Mac binary):

  • This version of STUI requires RO 2.2.18. This allows the Alerts window to not auto-scroll and allows users to control-click on X11 (a gesture used by the Interlocks display).

Internal changes:

  • Upgraded Mac version to latest opscore. This allows parsing Int and Bits keyword values expressed as hex.
  • Upgraded Mac version to latest actor dictionaries. This allows it to find the mcp keyword ffsCommandedOpen (which was formerly listed as ffsCommandedOn).

Version 0.1.1 2009-08-06

Visible changes:

  • Modified to adapt to a change in the alerts keyword from the alerts actor.

Internal changes:

  • Removed explicit setting of matplotlib "numerix" parameter because it is obsolete in the current release (0.99.0); users will have to make sure their matplotlib uses numpy and not some older, obsolete numeric package.

Bug fixes:

  • Added sound cue for serious alerts to the svn repository.

Version 0.1 2009-08-04

  • The first sdss version. Note that it was branched from STUI at around version 1.6, but I have merged some improvements in both directions.
