TUI:Tidbits:Error Log

TUI may occasionally print error messages to an error log. This can be very useful when reporting bugs or debugging your own code. The location of the error log depends on your platform and how you ran TUI:

unix

The error log is your terminal (unless you redirect stdout to a file).

If you prefer, you may have TUI manage a log file for you by running runtuiWithLog.py instead of runtui.py. The error log will be as described for MacOS X, but will be written to your home directory.

MacOS X

Error messages are written to a file "tuilog.txt" in your standard documents directory (~/Documents on an English system). This log file will be created even if there are no errors to report.

In addition, if TUI finds an existing "tuilog.txt", it renames it to "oldtuilog.txt" (any any existing "oldtuilog.txt" will be lost). Thus if you want to save a log file, please rename it to something other than "tuilog.txt" or "oldtuilog.txt".

If TUI cannot create the log file for any reason, then it writes messages to stderr, which goes to the console. Run /Applications/Utilities/Console to see the console.

Windows

Error messages are written to a file called "tui.log" in the folder containing "TUI.exe". The file will only be written if there is an error, and you will see a dialog box.
