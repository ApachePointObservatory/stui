Notes

To use, you must create an environment variable TCLLIBPATH that points to /net/tcllib. If you use tcsh and Chance Reschke's standard ENV setup, then you can add the following to ~/ENV/.cshrc.personal: 

setenv TCLLIBPATH "/net/tcllib"

Tcl/Tk Additions

  • snack 2.2.3: a sound library. See the on-line manual and home site. The Python interface tkSnack was also installed in /net/python. Installed 2004-01-08. Warnings:
    • Not all linux boxes seem have sound support. In particular, at the time of this writing, the two in the 3.5m observing room ("may" and "bonus") do not.
    • To hear or record sound you must be directly logged into a linux box. There is no way to transmit sound to an X server.

Installation Notes

Unless noted below, all software was built according to the included instructions and those instructions contained no surprises.

snack

The following changes were required:

  • ./configure requird flags: --with-tcl=/usr/lib --with-tk=/usr/lib
  • The Makefile had to be hand-edited to specify the destination prefix (specifying --prefix to ./configure did nothing).
  • Before testing, set environment variable TCLLIBPATH to include the directory in which the make was done.

The build sequence was:

cd snack2.2.3/unix
./configure --with-tcl=/usr/lib --with-tk=/usr/lib
# edit Makefile to change the following two lines;
# note: specifying --prefix to ./configure does not work:
SNACK_INSTALL_PATH = /net/tcllib    
SNACK_HINSTALL_PATH = /net/tcllib        
make
# test; modify TCLLIBPATH first:
setenv TCLLIBPATH .
make test
# install tcl code
make install
# install python module
cp ../python/tkSnack.py /net/python/lib/python2.3/site-packages
