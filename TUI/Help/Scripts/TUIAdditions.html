STUI:Scripts:STUIAdditions

STUI will look in two separate "STUIAdditions" directories for user scripts and extra STUI window code. One of these is for your private additions, the other is for additions shared with other users on the computer.

Scripts for the Scripts menu belong in a directory named Scripts in the STUIAdditions directory. For more information see Where To Put Scripts in the Scripting Manual.

Code to generate additional STUI windows can be put directly in a STUIAdditions directory. Writing such additions is described in the Programming guide.

The location of STUIAdditions

STUI looks for a directory named STUIAdditions in each of two locations: one unique to each user, the other shared by all users on a multi-user system.

Both STUIAdditions directories are optional and are not present by default. You can create the user version yourself. You may need special privileges to create the shared version.

The locations depend on which platform you are using:

  • unix: ~ (your home directory) and TUIRoot (the parent directory for STUI). STUIAdditions may be a symbolic link.
  • MacOS X: the private and shared application support directories. On US systems this is /Users/username/Library/Application Support and /Library/Application Support. STUIAdditions may be a symbolic link but it must not be an alias.
  • Windows: the private and shared application data directory. On US systems this is C:\Documents and Settings\username\Application Data and C:\Documents and Settings\All Users\Application Data. STUIAdditions must not be an alias.

In general, additions present in the user STUIAdditions directory mask identically named additions in the shared STUIAdditions directory, and both mask identically named built-in scripts and windows. Thus you should be a bit careful in naming additions. This is discussed in more detail in the scripting and programming manuals.
